Day 1 – Day 15: From Concept to Digital Solution

1. System Design, Modularity, and UI Architecture

In the initial stages, based on my preliminary descriptions, the AI generated designs that satisfied the basic functional requirements. However, these early outputs were fragmented and lacked systemic consistency across the application. Recognizing this limitation, I refined my prompts and directed the AI to restructure the project into a modular architecture consisting of separate HTML templates, a centralized CSS stylesheet, and a dedicated JavaScript file.

Rather than relying on isolated and disconnected web pages, I implemented a unified layout using a shared base.html template. This approach ensured consistency throughout the system while effectively applying the DRY (Don't Repeat Yourself) principle. As a result, development became more efficient, code duplication was significantly reduced, and future interface updates became much easier to manage.

Building each child page upon this shared foundation maintained both visual and structural consistency across the e-Portfolio. At the same time, individual sections—such as Education, Projects, and Contact—retained subtle stylistic distinctions that reflected their unique purposes, thereby improving usability, readability, and overall user experience.

2. Implementation, System Transition, and Critical Evaluation

After completing the frontend source code, the project progressed from interface design to functional implementation and testing. Although the AI initially suggested using Live Server in Visual Studio Code for rapid previewing, I deliberately chose to implement the application using Flask. This decision aligns more closely with my academic specialization in Health Technology and my professional interest in healthcare software and data-driven systems.

Integrating Flask enabled me to work within a dynamic web application environment and gain valuable practical experience with client–server architecture, routing, backend logic, and API development. This transition was a crucial milestone, transforming the e-Portfolio from a static website into an interactive and functional web application.

Following several rounds of debugging and refinement, I successfully developed a relatively complete system. Key features such as dark mode, a BMI calculator API, a contact form, and database integration significantly enhanced the application's functionality and professionalism.

Nevertheless, I recognize that the current implementation remains limited in scope. It does not yet include advanced backend capabilities such as robust authentication, comprehensive data analytics, role-based access control, or large-scale deployment considerations. To evolve this project into a fully scalable full-stack system, further development will be necessary.

Overall, this project has strengthened my technical skills in Flask, frontend development, database integration, and AI-assisted software engineering. More importantly, it has demonstrated the importance of critical evaluation, iterative refinement, and effective prompt engineering when collaborating with AI during the software development lifecycle.
